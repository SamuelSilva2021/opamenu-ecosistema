## Resumo das Entregas e Próximos Passos

Finalizamos com sucesso a implementação da **Gestão de Colaboradores**, permitindo agora o cadastro de entregadores internos e externos (parceiros) sem a necessidade de criar contas de usuário no sistema.

### O que foi entregue:

1.  **Backend (.NET)**:
    *   Nova entidade `CollaboratorEntity` e tabela `Production.Collaborators`.
    *   API completa (`/api/collaborators`) com Repository e Service.
    *   Atualização na entidade `Order` para vincular entregas a colaboradores.

2.  **Frontend (Flutter)**:
    *   Nova feature `Collaborators` completa (Clean Architecture + Riverpod).
    *   Tela de listagem e cadastro de colaboradores no menu lateral.
    *   Modal de seleção de entregador (`SelectDriverDialog`) atualizado para usar a nova lista.
    *   Correção de permissões de acesso (`USER` -> `USER_ACCOUNT`).

# Plano de Melhorias: Gestão de Colaboradores e Integração de Entregas

Este plano foca em refinar a interface de Colaboradores e garantir que a seleção de entregadores funcione perfeitamente com o fluxo de pedidos.

## 1. Melhorias na UI de Colaboradores (Frontend)
Vamos aprimorar a `CollaboratorsPage` para facilitar a gestão quando houver muitos cadastros.

### 1.1 Filtros e Busca
*   **Barra de Pesquisa**: Adicionar campo para filtrar por nome ou telefone.
*   **Tabs/Segmented Control**: Separar a lista em:
    *   **Todos**
    *   **Internos** (Funcionários com acesso ao sistema)
    *   **Externos** (Parceiros/Entregadores)
*   **Visual**: Melhorar os cards para destacar claramente o tipo de vínculo (ícones e cores distintas).

## 2. Refinamento da Seleção de Entregador (Frontend)
O modal `SelectDriverDialog` precisa ser mais funcional para operações rápidas na cozinha.

*   **Filtro Rápido**: Adicionar campo de busca no topo do modal.
*   **Ordenação Inteligente**: Listar primeiro os entregadores que estão "Livres" (futuro) ou ordenar alfabeticamente por padrão.
*   **Feedback Visual**: Melhorar o destaque do item selecionado.

## 3. Validação do Fluxo de Pedidos (Backend & Frontend)
Garantir que o vínculo do entregador está sendo persistido corretamente.

*   **Frontend**: Verificar se `ProductionController` está enviando o `driverId` corretamente no corpo da requisição `updateOrderStatus`.
*   **Backend**: O código analisado (`UpdatEOrderStatusAsync`) já possui a lógica para salvar o `DriverId`. Vamos testar se isso está ocorrendo na prática.

## 4. Plano de Execução
1.  **Refatorar `CollaboratorsPage`**: Implementar abas de filtro e busca.
2.  **Atualizar `SelectDriverDialog`**: Adicionar campo de busca.
3.  **Teste Integrado**: Realizar um ciclo completo de pedido Delivery (Pendente -> Pronto -> Saiu p/ Entrega com Driver -> Entregue) e verificar se o entregador ficou registrado no histórico.
